Lesson 0002: Build Wazuh and Log Analytics Security Dashboards

This lesson turns a working deployment into two dashboard sets: Wazuh for detections and OCI Log Analytics for cross-source correlation.

Module 1: Wazuh Detection Views

Create or open these Wazuh data views:

wazuh-alerts-*
oci-audit-*
oci-flow-*

Use wazuh-alerts-* for rule-driven detections. Use the raw OCI indices only when you need source-record context.

Module 2: Wazuh Filters

rule.id >= 100000 and rule.id <= 100099
rule.id >= 100100 and rule.id <= 100199
rule.groups: syscheck or rule.groups: sca
agent.name: (braavos or castelblack or kingslanding or meereen or winterfell) or rule.groups: windows

Save each filter as a saved search before adding charts. This keeps the dashboard explainable.

Module 3: Log Analytics Source Discovery

* | stats count by "Log Source" | sort -count

Source discovery is the first panel and the first troubleshooting step. Do not build a dashboard from assumed source names.

Module 4: Correlation Row

Build one row that answers a single security question: What denied network activity happened, which Wazuh alerts occurred in the same window, and which host or identity should be investigated?

Exercise

Use dashboards/log-analytics/oci-wazuh-dashboard-queries.json to create four saved searches: source inventory, VCN rejects by pair, OCI Audit by user/source, and Wazuh alert volume. Write a one-sentence dashboard title that states the investigation goal.

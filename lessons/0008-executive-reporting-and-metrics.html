Module 1: Separate Evidence from Metrics

Evidence is a concrete alert, raw event, or query result. A metric is a repeated measurement that shows direction. Do not present alert volume alone as security posture.

Module 2: Use a Small Metric Set

  • Coverage: percentage of scoped hosts with active Wazuh agents.
  • Cloud visibility: Audit and Flow ingestion gates green in the last review period.
  • Endpoint posture: top failing SCA checks and high-risk vulnerable hosts.
  • Detection health: high-value detections with owners and current dashboards.
  • Remediation: open and closed posture backlog items by control category.

Module 3: Build the Executive Story

What changed?
Which risks did we reduce?
Which sources increased coverage?
Which control gaps remain?
What decision or investment is needed?

Module 4: Keep Drilldown Nearby

Every executive metric should link back to a Wazuh view, a Log Analytics query, or a posture backlog item. Otherwise the metric cannot be audited.

Reporting rule: if a leader asks "show me the evidence," the dashboard owner should be one click away from the query or alert view.

Exercise

Create a one-slide executive summary for this demo: three metrics, two risks reduced, one remaining gap, and one decision request.

Deliverable: a leadership slide with metrics, evidence links, posture backlog movement, and the next decision needed.

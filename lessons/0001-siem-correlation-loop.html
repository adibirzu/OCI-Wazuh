Lesson 0001: Build a Wazuh and Log Analytics Correlation Loop

This lesson teaches one repeatable loop: choose a signal, normalize it, detect it in Wazuh, correlate it in OCI Log Analytics, and turn the result into a posture action.

Module 1: Pick the Signal

Start with one behavior that matters. For this demo, use denied network traffic. Wazuh receives normalized VCN Flow records and raises rule 100100. OCI Log Analytics shows the same network behavior beside Wazuh alert volume, host logs, and Windows/Sysmon events.

Module 2: Confirm the Sources

In Log Analytics, run source inventory first:

* | stats count by "Log Source" | sort -count

Do not assume source names from another tenancy. Confirm names such as OCI VCN Flow Unified Schema Logs, OCI Audit Logs, Windows Sysmon Events, and the custom Wazuh alert source before building widgets.

Module 3: Validate the Wazuh Detection

Run the live validation gate from the repository:

make validate-real-oci-logs

The gate creates real OCI Audit activity, generates denied network traffic, and waits for Wazuh rules 100000 and 100100. This proves the detection path before you build dashboard interpretation around it.

Module 4: Build the Correlation Widget

Use the query pack in dashboards/log-analytics/oci-wazuh-dashboard-queries.json. Start with the source inventory widget, then add VCN rejects, OCI Audit events by user/source, Wazuh alert volume, and Windows Sysmon network connections.

Exercise

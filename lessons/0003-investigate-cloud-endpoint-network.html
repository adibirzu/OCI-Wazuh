Analyst Drill

Lesson 0003: Investigate Cloud, Endpoint, and Network Signals

This lesson teaches a repeatable analyst drill: start from one alert, pivot through cloud and network context, then decide whether the right next action is tuning, investigation, or hardening.

AudienceSOC analyst, incident responder
Time35 minutes
Primary toolsWazuh alert view, raw OCI indices, Log Explorer
OutputThree-line investigation note

Investigation Storyboard

AlertRule, severity, agent, MITRE
Raw sourceAudit or Flow event fields
ContextOS, Sysmon, identity, network
DecisionTune, investigate, harden, accept
ProofVerification query or dashboard row

Evidence Checklist

Identity

Principal, user, source IP, domain account, or service identity.

Entity

Agent, host, VNIC, subnet, compartment, or Log Analytics entity.

Network

Source/destination IPs, ports, protocol, action, bytes, and packets.

Module 1: Start from Wazuh

Open wazuh-alerts-* and filter to either OCI Audit detections or VCN Flow detections:

rule.id >= 100000 and rule.id <= 100199

Record the timestamp, rule ID, agent or source, source IP, destination IP, principal, and MITRE technique if present.

Module 2: Pivot to Raw OCI Records

Use oci-audit-* for cloud control-plane activity and oci-flow-* for network records. Raw records help distinguish a broad detection from the exact API call or network flow that triggered it.

oci-audit-* fields: eventType, principalName, sourceIp, compartmentId
oci-flow-* fields: srcaddr, dstaddr, srcport, dstport, protocol, action, bytes, packets

Module 3: Pivot to Log Analytics

In Log Analytics, confirm whether the same time window has OS, Windows, Sysmon, or OCI service context. Start with source inventory, then run the saved searches from the dashboard query pack.

* | stats count by "Log Source" | sort -count

Module 4: Decide the Outcome

  • Tune: the event is expected and the rule needs better context.
  • Investigate: identity, endpoint, or network context is inconsistent with normal behavior.
  • Harden: the alert reveals a durable control gap such as excessive IAM rights, weak segmentation, or vulnerable software.
Rule of thumb: if you cannot name the owner or verification query, you do not yet have a posture action.

Lesson 0004: Turn Detections into a Security Posture Backlog

This lesson converts detection output into a practical improvement backlog. The goal is not more alerts; the goal is fewer unresolved control gaps.

Module 1: Classify the Gap

Every useful detection should map to one posture category:

  • Identity and access
  • Endpoint hardening
  • Network segmentation
  • Vulnerability management
  • Logging and detection coverage
  • Response process

Module 2: Write the Backlog Item

Title: Restrict unexpected rejected traffic to workload subnet
Evidence: Wazuh rule 100100 and Log Analytics VCN rejects by pair
Risk: possible scan or unapproved service exposure
Owner: network/security owner
Action: tighten NSG rule or validate approved scanner
Verification: reject spike no longer appears after change

Module 3: Attach ATT&CK Coverage

Use ATT&CK IDs when they help explain adversary behavior. Use control categories when they explain the actual remediation. Both are needed: ATT&CK frames the threat; the backlog item drives the fix.

Module 4: Close with a Query

Do not close a posture item with a meeting note. Close it with a repeatable query or Wazuh view that proves the gap changed.

Exercise

Create three backlog items: one for OCI Audit, one for VCN Flow, and one for Sysmon or Linux FIM. Each item must include evidence, owner, action, and verification query.

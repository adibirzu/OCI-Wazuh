Posture Management

Lesson 0004: Turn Detections into a Security Posture Backlog

This lesson converts detection output into a practical improvement backlog. The goal is not more alerts; the goal is fewer unresolved control gaps.

AudienceSecurity lead, control owner
Time30 minutes
Primary toolsWazuh views, Log Analytics queries, backlog tracker
OutputThree verified posture backlog items

Backlog Pipeline

SignalAlert or dashboard pattern
EvidenceWazuh and Log Analytics proof
Control gapIdentity, endpoint, network, vulnerability
OwnerNamed team or role
VerifyRepeatable query confirms change

Backlog Quality Bar

Evidence-backed

Every item points to a Wazuh rule, raw event, or Log Analytics query.

Owned

Every action has a team that can change policy, endpoint state, or network control.

Verifiable

Closure requires a query result, not only a meeting note or ticket comment.

Module 1: Classify the Gap

Every useful detection should map to one posture category:

  • Identity and access
  • Endpoint hardening
  • Network segmentation
  • Vulnerability management
  • Logging and detection coverage
  • Response process

Module 2: Write the Backlog Item

Title: Restrict unexpected rejected traffic to workload subnet
Evidence: Wazuh rule 100100 and Log Analytics VCN rejects by pair
Risk: possible scan or unapproved service exposure
Owner: network/security owner
Action: tighten NSG rule or validate approved scanner
Verification: reject spike no longer appears after change

Module 3: Attach ATT&CK Coverage

Use ATT&CK IDs when they help explain adversary behavior. Use control categories when they explain the actual remediation. Both are needed: ATT&CK frames the threat; the backlog item drives the fix.

Module 4: Close with a Query

Do not close a posture item with a meeting note. Close it with a repeatable query or Wazuh view that proves the gap changed.

Anti-pattern: "Reviewed by security" is not closure. "VCN rejects by pair no longer shows the unexpected source after NSG change" is closure.

Exercise

Create three backlog items: one for OCI Audit, one for VCN Flow, and one for Sysmon or Linux FIM.

Deliverable: each item must include control category, evidence, owner, action, verification query, and closure evidence.

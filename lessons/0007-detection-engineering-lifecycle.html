Lesson 0007: Detection Engineering Lifecycle

This lesson turns one useful signal into a managed detection. The lifecycle is source, normalize, detect, correlate, tune, measure, and retire or promote.

Module 1: Start with a Detection Hypothesis

Behavior: denied traffic spike against a monitored workload
Source: OCI VCN Flow Logs
Wazuh rule: 100100
Correlation: Log Analytics VCN rejects by source and destination
Posture question: is this approved scanning, misconfiguration, or attack discovery?

Module 2: Define Required Fields

Fields are the contract between ingestion, Wazuh rules, and dashboards. Missing fields create weak detections.

Minimum flow fields:
srcaddr, dstaddr, srcport, dstport, protocol, action, bytes, packets

Minimum audit fields:
eventType, principalName, sourceIp, compartmentId

Module 3: Tune with Evidence

Tuning is not hiding alerts. Tuning means adding known-good context, narrowing rule conditions, or changing severity after an evidence review.

Module 4: Promote the Detection

A detection is production-ready when the source is reliable, the rule fires on real data, the dashboard has an owner, and the response action is known.

Exercise

Pick one detection from the demo and write its lifecycle card: hypothesis, required fields, Wazuh rule/view, Log Analytics query, owner, tuning rule, and retirement condition.

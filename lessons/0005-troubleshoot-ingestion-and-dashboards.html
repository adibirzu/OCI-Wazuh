Lesson 0005: Troubleshoot Wazuh and Log Analytics Ingestion

This lesson teaches a deterministic troubleshooting path. The goal is to isolate whether a missing signal is a source problem, delivery problem, parser problem, dashboard problem, or expectation problem.

Module 1: Start with the Gate

Run the smallest validation that covers the missing signal:

make e2e
make goad-validate
make simulate-detections
make validate-real-oci-logs
make validate-opensearch-oci
make log-analytics-bridge

Do not debug dashboards until the ingestion gate proves the data exists.

Module 2: Locate the Break

  • No Wazuh alert: check decoder/rule deployment and logcollector inputs.
  • Wazuh alert exists, no raw OCI index: check OpenSearch sink configuration.
  • Raw index exists, no Log Analytics result: check OCI Logging, Unified Agent, Connector Hub, and Log Analytics propagation.
  • Log Analytics result exists, dashboard empty: check source names, quoted fields, filters, and time range.

Module 3: Use Source Inventory

When a Log Analytics dashboard is empty, run this before changing the query:

* | stats count by "Log Source" | sort -count

If the source name differs from the saved query pack, update the saved search for that tenancy.

Module 4: Keep a Failure Note

Symptom:
Expected signal:
Last green gate:
First red gate:
Suspected break:
Fix attempted:
Verification command:

This prevents random dashboard editing and makes the next failure faster to diagnose.

Exercise

Pick one empty dashboard widget. Identify the gate that should prove its data source, run or inspect that gate, then write the failure note above.

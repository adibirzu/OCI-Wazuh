Lesson 0006: Roll Out Wazuh and Log Analytics in an Enterprise

This lesson turns the demo into a reusable rollout plan. The company objective is to improve security posture without creating unmanaged logging cost, uncontrolled access, or dashboards nobody owns.

Module 1: Define Ownership

  • Security engineering: Wazuh rules, ATT&CK mapping, severity, and response playbooks.
  • Cloud platform: OCI Logging, Connector Hub, Streaming, Log Analytics onboarding, and IAM.
  • Endpoint team: Wazuh agents, Sysmon, SCA baselines, and patching workflow.
  • SOC: dashboards, triage, investigation notes, and escalation thresholds.

Module 2: Govern the Data

Before adding sources, decide retention, masking, access, cost owner, and allowed export paths. A useful dashboard that exposes sensitive identity or endpoint details still needs access control.

Module 3: Onboard by Detection Family

  1. Start with Wazuh alert health and Linux FIM/SCA.
  2. Add OCI Audit and VCN Flow ingestion.
  3. Add Windows and Sysmon for critical domains or labs.
  4. Bridge Wazuh alerts into Log Analytics.
  5. Create a posture backlog and review it weekly.

Module 4: Define Production Exit Criteria

Agents active: all scoped hosts
Ingestion: real Audit and Flow gates green
Dashboards: source inventory plus correlation panels reviewed
Security: no hardcoded secrets or public dashboard exposure
Operations: owner, runbook, retention, teardown, and upgrade path documented
Posture: at least three verified backlog remediations

Exercise

Write a one-page rollout charter for one business unit. Include sources, owners, access controls, dashboard consumers, and the first three posture backlog items.

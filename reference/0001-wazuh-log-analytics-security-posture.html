Wazuh + OCI Log Analytics Security Posture Reference

Core Model

Use Wazuh for endpoint agent telemetry, local detections, custom decoders/rules, FIM, SCA, vulnerability detection, Sysmon, and alert triage. Use OCI Log Analytics for cross-source search, entity-aware dashboards, OCI service logs, OS logs, Wazuh alert ingestion, and broader correlation views.

Telemetry Map

  • Wazuh alerts: Wazuh detections, rule IDs, agents, MITRE technique IDs, alert severity.
  • OCI Audit: API activity, principal, source IP, request target, response result.
  • VCN Flow Logs: source/destination IP, ports, protocol, action, bytes, packets.
  • Linux logs: syslog, secure logs, FIM, SCA, package inventory, vulnerability records.
  • Windows and GOAD: Security events, Sysmon events, Wazuh Windows agent alerts, SOC Fortress rule hits.

Correlation Keys

  • Entity: host name, agent name, instance OCID placeholder, or Log Analytics entity.
  • Identity: OCI principal, Windows account, Linux user, Wazuh alert user fields.
  • Network: source IP, destination IP, source port, destination port, action.
  • Time: align to a small window first, then widen only when source delivery latency requires it.
  • Technique: map detections to ATT&CK technique IDs for posture coverage and executive reporting.

Dashboard Pattern

  1. Run source discovery: * | stats count by "Log Source" | sort -count.
  2. Confirm field names for each source before saving widgets.
  3. Create source-family widgets: Wazuh alerts, OCI Audit, VCN Flow, Linux, Windows/Sysmon.
  4. Add correlation widgets: rejected flow plus Wazuh alert, privileged OCI action plus host change, Sysmon process plus network egress.
  5. Close with posture widgets: top ATT&CK techniques, top failing SCA checks, vulnerable hosts, unresolved high-severity alerts.

Operating Checks

  • make log-analytics-bridge validates source families and entities.
  • make validate-real-oci-logs proves real OCI Audit and VCN Flow records reach Wazuh.
  • make validate-opensearch-oci proves dedicated oci-audit-* and oci-flow-* indices contain records.
  • make goad-validate proves Windows/GOAD agents and Sysmon detections are active.
  • make down removes Wazuh demo resources and cleans Wazuh/Sysmon agents from reused GOAD hosts.
